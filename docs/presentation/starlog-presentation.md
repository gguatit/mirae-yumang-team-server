# STARLOG 프로젝트 발표 자료

**Team**: 미래유망팀
**프로젝트**: STARLOG - 학교 커뮤니티 플랫폼
**기술 스택**: Spring Boot 3.5.7 + Java 17 + Thymeleaf + MySQL

---

## 1. 프로젝트 한 줄 소개

STARLOG는 Spring Boot 기반의 학교 커뮤니티 플랫폼입니다. 학생들이 게시글을 작성하고, 댓글을 달고, 급식/시간표를 확인할 수 있는 통합 서비스입니다.

---

## 2. 기술 스택

| 구분 | 기술 |
|------|------|
| Backend | Spring Boot 3.5.7, Java 17, Spring MVC |
| Frontend | Thymeleaf, Vanilla JS, GSAP |
| Database | MySQL 8.0 (운영), H2 (개발) |
| Security | Spring Security, BCrypt, Cloudflare Turnstile |
| External API | NEIS (급식, 시간표, 학사일정) |
| Infra | Cloudflare Tunnel, systemd |

---

## 3. 아키텍처

### MVC 패턴
```
브라우저 -> Controller -> Service -> Repository -> Database
              |
            View (Thymeleaf)
```

**역할 설명:**
- **Controller**: 사용자 요청을 받아 적절한 Service 호출
- **Service**: 실제 비즈니스 로직 처리
- **Repository**: DB와 통신하여 데이터 저장/조회
- **View**: Thymeleaf로 HTML 페이지 렌더링

---

## 4. 핵심 기능 소개

### 4.1 회원 시스템
- 회원가입, 로그인, 로그아웃
- 비밀번호는 BCrypt로 암호화 저장
- 로그인 5회 실패 시 15분 잠금 (보안)

### 4.2 게시판
- 게시글 작성, 수정, 삭제 (이미지 첨부 가능)
- 댓글 및 대댓글
- 좋아요/싫어요 기능
- 실시간 새 게시글 알림 (5초 자동 갱신)

### 4.3 한글 검색
- 초성 검색: "ㅅㅍㄹ" -> "스프링" 검색
- 영타 변환: "dkssud" -> "안녕" 검색
- 일반/고급 검색 모드 지원

### 4.4 학교 정보 페이지
- 근명고등학교 급식 정보 조회
- 학년/반 선택 시간표
- 이번 달 학사일정
- 모든 데이터는 NEIS API로 실시간 조회

### 4.5 마이페이지
- 프로필 이미지 변경
- 자기소개 수정
- 내 게시글 목록
- 회원 탈퇴

## 5. 시연 플로우 (데모 순서)

1. **홈페이지 접속** (https://starlog.c01.kr)
   - Spline 3D 인터랙티브 배경 확인
   
2. **회원가입**
   - 사용자명, 비밀번호, 이메일 입력
   - Cloudflare Turnstile 봇 방지 체크
   
3. **게시글 작성**
   - 제목, 내용, 이미지 첨부
   
4. **검색 기능**
   - 일반 검색: "스프링"
   - 초성 검색: "ㅅㅍㄹ"
   - 영타 검색: "tkdls"
   
5. **학교 정보 페이지** (/school)
   - 급식 정보 확인
   - 학년/반 선택 후 시간표 조회
   - 학사일정 확인

6. **마이페이지**
   - 프로필 이미지 변경
   - 내 게시글 확인

## 6. 보안 기능

- 비밀번호 BCrypt 암호화
- XSS 방지 (Jsoup으로 HTML 제거)
- 파일 업로드 검증 (확장자, MIME, 크기)
- 브루트포스 방어 (5회 실패 시 잠금)
- Cloudflare Turnstile (봇 방지)
- HTTPS (Cloudflare Tunnel)
- 세션 보안 (Secure Cookie, HttpOnly, SameSite)

## 7. 운영 환경

- **서버**: Ubuntu + systemd
- **도메인**: starlog.c01.kr (Cloudflare Tunnel)
- **자동 시작**: 서버 재부팅 시 서비스 자동 실행
- **모니터링**: systemd + journalctl 로그 확인

## 10. Q&A 예상

**Q. 왜 Spring Boot를 선택했나요?**
A. Java 기반으로 안정적이고, 학교에서 배우는 내용과 연계하기 좋습니다. 또한 Spring의 다양한 기능(보안, DB 연동)을 쉽게 사용할 수 있습니다.

**Q. 학교 정보는 어떻게 가져오나요?**
A. 교육부 NEIS API를 사용합니다. 급식, 시간표, 학사일정을 무료로 제공하는 공공 API입니다.

**Q. 서버 운영 비용이 드나요?**
A. 학교 서버에 Ubuntu를 설치하고, Cloudflare Tunnel로 외부에 공개해서 추가 비용이 없습니다.

**Q. 실시간 업데이트는 어떻게 구현했나요?**
A. WebSocket 대신 5초 간격 폴링 방식을 사용했습니다. 구현이 간단하고 충분히 빠릅니다.
